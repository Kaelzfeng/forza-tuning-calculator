\# UI-Max Skill



You are improving a Forza tuning calculator website.



\## Core Goals



Create UI that feels:



\- premium

\- fast

\- gaming-oriented

\- mobile-first

\- readable

\- SEO-safe

\- easy to maintain



Visual direction:



\- Forza / Gran Turismo inspired

\- dark glassmorphism

\- neon accent only where useful

\- compact dashboard layout

\- clear tuning panels

\- strong mobile usability



\## Hard Rules



Before editing files:



1\. inspect existing files first

2\. explain what will change

3\. modify the minimum number of files

4\. avoid full rewrites

5\. preserve existing routes and logic



Never:



\- rewrite the whole app

\- delete working logic

\- install dependencies without permission

\- change Directus schema

\- change build config unless asked

\- use random UI libraries



\## Windows Rule



The environment is Windows PowerShell.



Use PowerShell-compatible commands only.



Do not use:



\- ls -la

\- grep

\- sed

\- awk

\- rm -rf

\- cp -r



Use:



\- Get-ChildItem

\- Select-String

\- Remove-Item

\- Copy-Item

\- New-Item



\## Vue Rules



Use:



\- Vue 3

\- Composition API

\- reusable components

\- clear props

\- simple state

\- computed values for derived data



Avoid:



\- giant components

\- duplicated UI blocks

\- huge reactive objects

\- business logic inside templates



\## Tailwind Rules



Use TailwindCSS utility classes.



Prefer:



\- rounded-2xl

\- backdrop-blur

\- bg-white/5

\- border-white/10

\- shadow-xl

\- responsive grids

\- sticky mobile actions where useful



Avoid:



\- excessive animations

\- low-contrast text

\- hardcoded random colors everywhere

\- layout shifts



\## Calculator UI Requirements



A good tuning calculator UI should include:



\- input panel

\- result panel

\- tuning recommendation cards

\- explanation area

\- beginner/advanced toggle

\- mobile-friendly sliders

\- clear reset button

\- export/share button later



\## Output Style



After changes, always report:



\- files changed

\- what changed

\- why it changed

\- how to test

\- risk level
